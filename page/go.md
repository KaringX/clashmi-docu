# Clash Mi 服务商对接指南

## 深度集成概述

Clash Mi 可对接常见服务商（机场）的面板系统，比如 v2board、xboard、sspanel。

### 一、改善用户体验
- **面板登录**：使用用户名、密码直接登录，无需手动输入订阅地址。
- **订阅代理**：内置代理机制，突破网络封锁。

### 二、提升品牌认知
- 自定义 Logo、客服入口，展现服务商品牌。
- App 消息推送直达用户。
- 套餐到期提醒，减少用户流失。

### 三、防封锁
- 多域名轮询 + 订阅代理 + 消息推送，确保用户在复杂网络环境下仍可正常使用。

### 三、对接简单
- 仅需设置几个选项即可完成对接。

## 对接步骤

### 第一步：登录服务商管理后台

访问注册 [KaringX服务商管理后台](https://harry.karing.app/auth/login)

### 第二步：创建服务商

如果您还没有创建服务商，进入后台后：

1. 填写 **服务商名称**（纯文字，不能包含空格和特殊符号）
2. 填写 **主页链接**（您的官网地址）
3. 选择 **服务地域**（CN / RU / IR / VN / EN）
4. 选择 **面板系统类型**
5. 提交创建

创建成功后，系统会生成一个唯一通行码（uniqid），用于标识您的服务商身份。

### 第三步：进入 MI 配置页面

在基础信息页面，右侧点击"Mi App"卡片的「开始对接」按钮，进入 MI 配置页面。

### 第四步：填写 MI 配置

#### 必填字段

| 字段         | 说明                                                            |
| ------------ | --------------------------------------------------------------- |
| **面板类型** | 选择您的面板系统（v2board / xboard / sspanel）                  |
| **面板域名** | 您的面板服务域名，**纯域名不带 `http://`**，每行一个，最多 5 个 |

> 面板域名用于 App 验证您的面板地址。至少填写 1 个域名。不支持 IP 地址。

#### 可选字段

| 字段            | 说明                               | 继承规则                            |
| --------------- | ---------------------------------- | ----------------------------------- |
| 服务商名称      | 显示在 App 中的名称                | 留空则继承基础信息的名称            |
| 面板版本        | 如 `1.7.4.1681`                    | 留空则客户端自行判断  (v2board小于1.7版本得填)             |
| User-Agent      | 客户端访问面板时使用的 UA          | -                                   |
| 开启硬件指纹    | 启用后限制设备数量（基于硬件指纹） | FAQ:[硬件标识符](/guide/faq#x-hwid) |
| 主页 URL        | 服务商官网                         | 作为 App 内的跳转链接               |
| 订阅频道URL        | 订阅频道                    | 套餐购买入口                  |
| 在线客服 URL    | 客服/工单页面                      | 留空则继承基础信息的客服链接        |
| 服务商 Logo URL | App 内展示的服务商 Logo            | 需要 **logo 显示** VIP 权益         |
| App 图标 URL    | 自定义 App 图标                    | 需要 **自定义图标** VIP 权益        |


### 第五步：保存配置

填写完成后点击保存。用户在登录时输入对应通信码及会更新配置。

## VIP 权益

| 权益            | 功能                     | 说明                         |
| --------------- | ------------------------ | ---------------------------- |
| Logo 显示       | 自定义 Logo URL 字段可用 | App 中显示您自己的品牌 Logo  |
| 自定义通行码    | 可添加自定义通行码       | 让用户使用您专属的通行码绑定 |
| 消息推送        | 消息推送功能可用         | 向所有 App 用户推送通知      |
| 自定义 App 图标 | 自定义图标 URL 字段可用  | App 图标变为您的品牌图标     |

## 常见问题

**Q: 面板类型列表中没有我的系统怎么办？**
A: 请联系开发人员添加您的面板系统支持。Telegram: @ovowe @ElonWang

**Q: 域名填写有什么要求？**
A: 必须为有效域名，每行一个，不支持 IP 地址，不支持带 `http://` 协议前缀。

**Q: 自定义 App 图标什么时候生效？**
A: 新安装 App 或重新下载后生效，详情请咨询管理员。
