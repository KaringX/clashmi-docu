# Clash Mi 服务商安全对接指南

## 为什么需要服务商对接

近期，海外直连入口被封、中国大陆中转线路被通报和拔线的风险持续增加。对服务商来说，真正危险的往往不只是代理协议本身，而是订阅链接、节点入口和面板信息被泄露。

一旦用户将订阅分享、倒卖，或订阅被第三方客户端、爬虫、采集平台收集，同一入口下的其他用户也可能受到影响。严重时，整条中转线路可能被通报、封禁或拔线。

目前常见的防护方式包括：

| 方案 | 优点 | 局限 |
| --- | --- | --- |
| 封端 | 降低订阅被第三方程序和爬虫收集的概率 | 用户必须使用官方客户端，第三方客户端用户会感到不便 |
| 私有 DNS | 节点域名不再依赖公共 DNS 解析 | 客户端配置复杂，小白用户不友好，仍可能因配置或传播造成泄露 |
| Clash Mi 服务商对接 | 使用 App 内登录、订阅代理、多域名轮询和消息推送降低入口暴露影响 | 不能保证 100% 避免通报，但能减少订阅和入口裸奔带来的连锁风险 |

Clash Mi 服务商对接的目标不是承诺绝对防封，而是在不明显牺牲用户体验的前提下，帮助服务商降低订阅泄露、入口泄露和域名不可用带来的影响。

## 功能价值

### 一、降低订阅泄露风险

- 用户可通过✈️编码/别名和面板账号直接在 Clash Mi 内登录，无需手动复制订阅链接。
- 客户端内置订阅代理机制，减少订阅地址直接暴露在公共网络环境中的机会。
- 可配置 User-Agent、硬件指纹等限制条件，降低异常客户端和滥用行为的风险。

### 二、兼顾安全和用户体验

- 用户不需要自行配置私有 DNS、复杂规则或额外插件。
- 支持常见面板系统，适合 v2board、xboard、sspanel 等服务商快速接入。
- 对小白用户更友好，同时比纯手动订阅更容易统一管理入口。

### 三、提升服务可用性

- 支持多面板域名配置，客户端可根据配置进行轮询。
- 配合订阅代理和 App 消息推送，在域名异常、入口调整或线路变更时更容易通知用户。
- 套餐到期提醒、客服入口和官网入口可直接在 App 内展示，减少用户流失。

### 四、增强品牌认知

- 可展示服务商名称、Logo、官网、客服入口和订阅频道。
- VIP 服务商可使用自定义✈️编码/别名和自定义 App 图标，让用户更容易识别官方入口。

## 适合哪些服务商

Clash Mi 服务商对接适合以下场景：

- 希望减少订阅链接在第三方客户端、爬虫或采集平台中传播。
- 希望降低单个用户泄露订阅后影响整组入口的风险。
- 正在使用或计划使用中国大陆中转、优质入口、多域名容灾等方案。
- 希望给小白用户提供比私有 DNS 更简单的使用方式。
- 希望在不完全封闭生态的前提下，提高官方客户端入口的使用比例。

> 说明：任何客户端方案都无法保证 100% 避免通报、封禁或拔线。该功能的重点是降低泄露概率、缩小泄露影响，并提高入口调整后的触达能力。

## 对接流程

### 第一步：登录服务商管理后台

访问并注册 [KaringX 服务商管理后台](https://harry.karing.app/auth/login)。

### 第二步：创建服务商

如果您还没有创建服务商，进入后台后填写以下信息：

1. **服务商名称**：纯文字，不能包含空格和特殊符号。
2. **主页链接**：您的官网地址。
3. **服务地域**：CN / RU / IR / VN / EN。
4. **面板系统类型**：选择当前使用的面板系统。
5. 提交创建。

创建成功后，系统会生成一个唯一✈️编码/别名（uniqid），用于标识您的服务商身份。用户后续可通过该✈️编码/别名在 Clash Mi 内绑定服务商。

### 第三步：进入 MI 配置页面

在基础信息页面，点击右侧「Mi App」卡片中的「开始对接」按钮，进入 MI 配置页面。

### 第四步：填写 MI 配置

#### 必填字段

| 字段 | 说明 |
| --- | --- |
| **面板类型** | 选择您的面板系统，如 v2board / xboard / sspanel |
| **面板域名** | 您的面板服务域名，纯域名不带 `http://` 或 `https://`，每行一个，最多 5 个 |

> 面板域名用于 App 验证您的面板地址。至少填写 1 个域名，不支持 IP 地址。

#### 可选字段

| 字段 | 说明 | 继承规则或备注 |
| --- | --- | --- |
| 服务商名称 | 显示在 App 中的名称 | 留空则继承基础信息的名称 |
| 面板版本 | 如 `1.7.4.1681` | 留空则客户端自行判断；v2board 小于 1.7 版本建议填写 |
| User-Agent | 客户端访问面板时使用的 UA | 可用于区分官方客户端请求 |
| 开启硬件指纹 | 启用后限制设备数量 | FAQ：[硬件标识符](/guide/faq#x-hwid) |
| 主页 URL | 服务商官网 | 作为 App 内跳转链接 |
| 订阅频道 URL | 服务商公告或订阅频道 | 用于引导用户查看服务商动态 |
| 在线客服 URL | 客服或工单页面 | 留空则继承基础信息的客服链接 |
| 服务商 Logo URL | App 内展示的服务商 Logo | 需要 **Logo 显示** VIP 权益 |
| App 图标 URL | 自定义 App 图标 | 需要 **自定义 App 图标** VIP 权益 |

### 第五步：保存配置并引导用户登录

填写完成后点击保存。用户在 Clash Mi 中输入对应✈️编码/别名，并使用面板账号登录后，即可获取服务商配置。

建议服务商将✈️编码/别名、登录方式和官方客户端下载地址放在官网、公告或客服自动回复中，减少用户继续传播原始订阅链接。

| 权益 | 功能 | 说明 |
| --- | --- | --- |
| Logo 显示 | 自定义 Logo URL 字段可用 | App 中显示您自己的品牌 Logo |

## 使用建议

- 尽量引导用户使用✈️编码/别名和面板账号登录，减少原始订阅链接传播。
- 面板域名建议准备多个可用域名，避免单域名异常后影响全部用户。
- 发生入口调整、域名变更或线路维护时，优先通过 App 消息推送通知用户。
- 如果已经采用封端或私有 DNS，可将 Clash Mi 服务商对接作为更易用的客户端入口补充。
- 对高风险入口和中转线路，仍建议配合服务端风控、订阅权限控制和异常访问监控一起使用。

## 常见问题

**Q: Clash Mi 服务商对接能完全避免通报或封 IP 吗？**

A: 不能。它的作用是降低订阅链接和入口信息泄露的概率，并在入口变化时提升用户触达能力。通报、封禁和拔线还会受到线路来源、入口暴露、用户行为、DNS、服务端配置等因素影响。

**Q: 已经做了封端，还需要对接 Clash Mi 吗？**

A: 如果您希望用户通过 Clash Mi 获得更好的 App 体验、消息推送、多域名容灾和品牌展示，可以继续对接。封端偏向限制客户端来源，Clash Mi 对接偏向提供官方化入口和统一配置能力。

**Q: 已经使用私有 DNS，还需要这个功能吗？**

A: 私有 DNS 对部分用户配置成本较高。Clash Mi 对接可以让用户通过 App 登录获取配置，减少手动配置环节，更适合小白用户。

**Q: 面板类型列表中没有我的系统怎么办？**

A: 请联系开发人员添加您的面板系统支持。Telegram: @ovowe @ElonWang

**Q: 域名填写有什么要求？**

A: 必须为有效域名，每行一个，不支持 IP 地址，不支持带 `http://` 或 `https://` 协议前缀。

**Q: 自定义 App 图标什么时候生效？**

A: 新安装 App 或重新下载后生效，详情请咨询管理员。
